Explanation of the Process

For this project, I wanted to demonstrate how to run existing implementations of volume visualization methods like "volume ray casting". Instead of writing the code myself, I looked for available implementations on GitHub. Here's a step-by-step explanation of how I achieved this:

  1. Finding the GitHub Project:
    • I conducted a search on GitHub for projects related to volume ray casting
    • One of the repositories that caught my attention was the following: https://github.com/RolandR/VolumeRayCasting
    • This repository provided an implementation of volume ray casting, which I decided to use for my project.
  2. Forking the Repository:
    • In order to have my own copy of the repository to work with, I clicked on the "Fork" button on the top-right corner of the repository page.
    • This created a copy of the repository under my GitHub account.
  3. Enabling GitHub Pages:
    • To host my project's webpage, I needed to enable GitHub Pages for my forked repository.
    • I navigated to the settings of my forked repository and found the "GitHub Pages" section.
    • From there, I selected the branch I wanted to use for GitHub Pages (in this case, "master").
    • Additionally, I specified the root folder where my webpage files were located.
  4. Creating the Webpage:
    • Inside my forked repository, I created an HTML file called "index.html" that would serve as the main webpage for running the volume ray casting code.
    • I included the necessary code from the original repository and made any modifications required to integrate it into my webpage.
    • I also took the opportunity to improve the design of the webpage by adding basic CSS styles to enhance its appearance.
  5. Committing and Pushing Changes:
    • Once I was satisfied with the changes I made to the repository, I committed the modifications.
    • After committing, I pushed the changes to my forked repository on GitHub, ensuring that all the necessary files were included.
